# scala-cli-sbt-plugin

POC: an sbt 1.x plugin built with scala-cli, with scripted tests driven by a separate scala-cli "build" project.

## Layout

- `plugin/` ‚Äî the sbt plugin source (Scala 2.12, scala-cli). Publishes to Maven Central as `org.polyvariant:scala-cli-sbt-plugin-poc_2.12_1.0`.
- `plugin/src/sbt-test/<group>/<name>/` ‚Äî sbt scripted tests.
- `plugin/resources/sbt/sbt.autoplugins` ‚Äî required so sbt 1.x discovers the AutoPlugin (this file is normally generated by sbt's compiler plugin).
- `build/` ‚Äî a separate scala-cli project that publishes the plugin into a scratch local Maven repo and invokes `sbt.scriptedtest.ScriptedRunner` from `org.scala-sbt::scripted-sbt-redux`.

## Run scripted tests locally

```
scala-cli run build                            # all tests (default subcommand: scripted)
scala-cli run build -- scripted simple/basic
PLUGIN_VERSION=1.2.3 scala-cli run build
scala-cli run build -- scripted --no-publish   # skip local publish, expect remote artifact
```

The runner shells out to `scala-cli publish local --maven-local --m2-home <scratch>` to publish the plugin under `<organization>/<name>_2.12_1.0/<version>/` (sbt's Maven cross-paths convention), then spawns sbt with `-Dsbt.repository.config` pointing at the scratch repo.

## Publish to Maven Central

```
scala-cli run build -- publish --signer gpg --gpg-key <KEY_ID>
```

## Versioning

`Plugin.scala` uses `//> using publish.computeVersion git:dynver`:

- On a `v*` tag ‚Üí the tag (e.g. `v0.1.0` ‚Üí `0.1.0`).
- On any other commit ‚Üí `<lastTag>+<n>-<sha>-SNAPSHOT` (e.g. `0.1.0+3-abc1234-SNAPSHOT`).

scala-cli routes `-SNAPSHOT` versions to the Central snapshots host automatically.

## CI pipeline (`.github/workflows/ci.yml`)

Three jobs, chained via `needs:`:

1. **`test`** ‚Äî runs on every push and PR. Executes `scala-cli run build -- scripted` against the working tree (publishes the plugin into a scratch m2 and runs the scripted tests).
2. **`publish`** ‚Äî runs on `main` pushes and `v*` tags, after `test` passes. Imports the GPG key from `PGP_SECRET`, configures Sonatype credentials, and runs `scala-cli run build -- publish --signer gpg --gpg-key DB33C5B9DA1A245B`. On `main` this produces a `-SNAPSHOT` artifact; on a `v*` tag, a release.
3. **`verify-release`** ‚Äî runs on `v*` tags only, after `publish`. Polls Maven Central until the released `.pom` is reachable (up to ~15 minutes), then runs `scala-cli run build -- scripted --no-publish` with `PLUGIN_VERSION=<tag>` so the scripted tests resolve the *released* artifact from Central rather than the scratch repo.

Required secrets: `PGP_SECRET`, `SONATYPE_USERNAME`, `SONATYPE_PASSWORD`.
